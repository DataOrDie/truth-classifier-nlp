# Hybrid Transformers Journey

Experiment log for the next phase — pushing past the `all-mpnet-base-v2` stacking ceiling (macro_f1 = **0.6428**). The transformer fine-tuning phase established that DeBERTa-v3-small text-only peaks around 0.6205. This phase focuses on better fusion of language signal with metadata, more training data coverage via k-fold, and larger models.

The transformer trails stacking. The two most likely causes:
1. **Single split**: the transformer is trained on ~6,400 samples and validated on ~900 — much less data than the stacker sees across folds.
2. **Metadata kept separate**: the MLP branch learns metadata signals independently; DeBERTa's attention mechanism cannot cross-attend between text tokens and speaker/party context.

## Experiment Plan

### Experiment 1 — Text Formatting: prepend metadata into the input string

**Hypothesis:** Injecting speaker/party/subject directly into the tokenized input lets DeBERTa's self-attention cross-attend between the claim text and the speaker's identity. This is more natural than a parallel MLP branch, and gives the language model a chance to apply its world knowledge about political figures.

**Input format:**
```
"speaker: john edwards | party: democrat | subject: health-care | {statement text}"
```

**Why this might beat the MLP hybrid:**
- The MLP branch sees metadata as 4 float features (aggregated true-rates). The formatted string gives the model raw tokens: the actual speaker name, party name, and topic words — which the pre-trained model already has representations for.
- No architecture change needed — it is `transformer.py` with a different tokenizer input. No second optimizer, no concat layer, no BatchNorm.
- Proven pattern in NLP competitions: prepended context consistently outperforms late-fusion on small datasets.

**What to watch:**
- Max token length: median statement is ~20 tokens. Adding the prefix adds ~10–15 tokens. `max_length=128` is still sufficient.
- Rare/unknown speakers: use the raw name string (not the true-rate). The model may generalise from partial name matches.
- Threshold tuning still required — same grid search on val set.

**Script:** new `transformer_textformat.py` based on `transformer.py`, single-split, same LLRD config as Run 7.

---

### Experiment 2 — K-Fold Ensemble of the Best Text Variant

**Hypothesis:** The transformer is underfitting due to seeing only ~72% of the training data per run. Stratified 5-fold gives every sample a turn in the training set, and ensembling 5 checkpoints reduces variance.

**How it works:**
- 5-fold stratified CV. Each fold trains on 80% (~7,160 rows), validates on 20% (~1,790 rows).
- Each fold saves its best checkpoint (by val macro_f1).
- OOF predictions are collected across all folds — this is a full unbiased estimate of training-set performance.
- At test time: average softmax probabilities from all 5 fold models.

**Expected gain:** +0.01–0.03 macro_f1 over the single-split version. K-fold is almost always worth the compute cost for models this size.

**Compute cost:** ~5× the single-run training time. On RTX 5070: roughly 10–15 min total for DeBERTa-v3-small.

**Script:** `transformer_kfold.py`. Apply to whichever format (plain text or formatted) performs best in Experiment 1.

---

### Experiment 3 — Late Fusion: add transformer OOF probas to the mpnet stacker

**Hypothesis:** Even if the fine-tuned transformer underperforms stacking alone, its error profile differs enough to add signal when fused. This is Option C from `6.Transformers-Journey.md`, now with the k-fold OOF probas available.

**How it works:**
- Take the OOF probability column from Experiment 2 (or any transformer variant).
- Add it as an extra base-model column in `stacking.py` alongside the LR/RFC/LGBM/CatBoost OOF columns.
- The meta-learner (Logistic Regression) re-learns the blend weight.

**Why it's low-risk:** degrades gracefully. If the transformer OOF adds no signal, the meta-LR assigns it near-zero weight.

**Script:** modify `stacking.py` to accept an optional extra OOF column file. No retraining of base models needed.

---

### Experiment 4 — DeBERTa-v3-base (model upgrade)

**Hypothesis:** DeBERTa-v3-small has 12 layers / 768-dim hidden. DeBERTa-v3-base has 12 layers / 768-dim hidden but more parameters due to a larger intermediate FFN and more attention heads. On LIAR-style fact-checking benchmarks the gap is typically 1–3 F1 points.

**Feasibility on RTX 5070 (12 GB VRAM):**

| Model | Params | VRAM @ bs=16 | VRAM @ bs=8 |
|-------|--------|--------------|-------------|
| deberta-v3-small | 86M | ~4–5 GB | ~3 GB |
| deberta-v3-base | 184M | ~7–8 GB | ~5 GB |

Base fits comfortably. Use `batch_size=16, max_length=128, LLRD_FACTOR=0.9`.

**Script:** swap `MODEL_NAME = "microsoft/deberta-v3-base"` in the best-performing script from Experiments 1–2.

---

### Experiment 5 — LoRA Fine-Tuning of a 7B LLM (Mistral / Llama-3)

**Hypothesis:** A 7B model's language understanding is qualitatively better at detecting deceptive framing, hedged language, and implausible claims — signals that are hard to learn from 8,950 examples with a 86M model.

**Feasibility on RTX 5070 (12 GB VRAM):**
- 4-bit quantization via `bitsandbytes` (NF4) + LoRA (`peft`) fits a 7B model in ~8–9 GB VRAM.
- Training with LoRA (r=8, alpha=16, target: q_proj + v_proj) adds ~4M trainable parameters — fast and safe from overfitting.
- Libraries: `transformers`, `peft`, `bitsandbytes`, `trl` (for `SFTTrainer`).

**Candidate models:**
- `mistralai/Mistral-7B-v0.1` — strong on classification, well-supported
- `meta-llama/Meta-Llama-3-8B` — slightly larger but better instruction following

**Input format (same as Experiment 1 with a chat prompt):**
```
"Classify this political statement as TRUE or FALSE.
Speaker: {speaker} | Party: {party} | Subject: {subject}
Statement: {statement}
Answer:"
```

**Output:** single token `TRUE` or `FALSE`. Extract logit for each and convert to probability.

**What to watch:**
- Training time per epoch: ~20–30 min on RTX 5070 with bs=4, gradient accumulation=4.
- Risk of overfitting: 8,950 samples is genuinely small for a 7B model even with LoRA. Use early stopping aggressively (patience=1 epoch).
- This is the highest ceiling option but also highest variance.

**Result vs. baselines:** worse than text-only Option A Run 7 (0.6205) and well below the stacking ceiling (0.6428). Adding the MLP branch hurt rather than helped.

**Analysis:**

The run revealed a clear failure mode — the model plateaued at val macro_f1 = **0.5836 across all 3 epochs** (best checkpoint is epoch 1, epochs 2 and 3 never improve). The training loss oscillates rather than decreasing meaningfully after epoch 1 (0.7282 → 0.6863 → 0.6855). Val proba range stays narrow at [0.22, 0.75] throughout — the model is not moving to a confident calibration.

Likely causes:
1. **MLP branch gradient interference.** The metadata MLP has a much lower-dimensional, lower-loss signal than the transformer text path. With LLRD disabled (`FREEZE_EPOCHS=0`) and no warm-up phase for the new heads, the combined gradient may be destabilising early DeBERTa layers before the text path has converged.
2. **Parallel heads fight for the same logit space.** Concatenating CLS (768-dim) with MLP output (16-dim) and projecting through a single Linear(784→2) means the gradient from the MLP path has outsize influence on the shared head weights — 16/784 ≈ 2% of the feature space but potentially much more of the gradient early in training.
3. **True-rate features are leaky in the wrong direction.** The metadata MLP receives `speaker_true_rate` as a float — it can overfit this directly rather than learning language patterns, which may suppress the text path.

**Takeaway:** the parallel MLP architecture is not the right fusion strategy for this dataset size. Move directly to **Experiment 1 (text formatting)** — inject metadata as raw tokens in the input string instead of a separate branch. This lets DeBERTa's existing attention mechanism do the fusion without competing gradient paths.

However, the val_f1 curve is clearly still climbing at epoch 3 (0.3932 → 0.5930 → 0.6194) and the training loss (0.6264) has not converged. **The model ran out of epochs.** Two factors contribute:

1. **Freeze epoch 1 is wasted.** With formatted input, the CLS representation at epoch 1 is shaped by the new prefix tokens (speaker/party/subject in positions 1–8). The frozen classifier head cannot adapt to this shifted representation — val proba range is [0.5031, 0.5112], essentially random. The freeze strategy that helped plain-text runs makes formatted-text runs lose a full epoch of learning.
2. **Three epochs is too short for this task with LLRD.** The LR range of [9.57e-06, 2.00e-05] gives the lower layers almost no movement in 2 active epochs. With more epochs the lower layers can slowly shift their representations toward political credibility signals.

**Next experiment:** run `transformer_textformat.py` with `FREEZE_EPOCHS=0, EPOCHS=5`. Removing the freeze gives 3 more effective LLRD epochs vs. the current run. If epoch 3 val_f1 was 0.6194, epochs 4–5 with a well-warmed LLRD should push further.

### Experiment 3 — Late Fusion: add transformer OOF probas to the mpnet stacker

**Hypothesis:** Even if the fine-tuned transformer underperforms stacking alone, its error profile differs enough to add signal when fused. This is Option C from `6.Transformers-Journey.md`, now with the k-fold OOF probas available.

**How it works:**
- Take the OOF probability column from Experiment 2 (or any transformer variant).
- Add it as an extra base-model column in `stacking.py` alongside the LR/RFC/LGBM/CatBoost OOF columns.
- The meta-learner (Logistic Regression) re-learns the blend weight.

Here's the two-step workflow:

  Step 1 (~2 min) — extract holdout/test probas from the saved fold checkpoints:
  python src/training/transformer_kfold_extract.py
  This saves ho_proba.npy, ho_idx.npy, and test_proba.npy to models/transformer_kfold/.

  Step 2 — run stacking as usual:
  python src/training/stacking.py
  It auto-detects the transformer files. If all three exist it prints [Late fusion] Transformer k-fold artifacts found —
  adding as 5th base model and includes the transformer OOF as a 5th column in the meta-LR. If they're missing it falls back
   to standard 4-model stacking silently.

  The meta-LR will assign the transformer whatever weight is optimal — if the signal is redundant with what XGB/CAT already
  see it'll get a near-zero coefficient, so there's no risk of hurting the existing 0.6428.

### Experiment 4 — DeBERTa-v3-base (model upgrade)

**Hypothesis:** DeBERTa-v3-small has 12 layers / 768-dim hidden. DeBERTa-v3-base has 12 layers / 768-dim hidden but more parameters due to a larger intermediate FFN and more attention heads. On LIAR-style fact-checking benchmarks the gap is typically 1–3 F1 points.
